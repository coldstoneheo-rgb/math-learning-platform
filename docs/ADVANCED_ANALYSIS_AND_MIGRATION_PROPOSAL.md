# 고급 수학 학습 분석 및 레거시 데이터 통합 제안서

이 문서는 수학 학습 플랫폼이 단순한 문제 풀이 채점 및 코멘트 요약을 넘어, **초개인화된 분석/컨설팅 시스템**으로 진화하기 위한 전략적 방향성을 제시합니다.

---

## 1. 학습 분석 퀄리티 향상을 위한 다각화 전략 (Beyond OCR & Comments)

현재의 문제 풀이 스캔 이미지와 교사 코멘트 분석은 훌륭한 기반입니다. 여기서 분석의 질을 기하급수적으로 끌어올리기 위해 **결과 중심 분석에서 '과정(Process) 및 인지 상태' 분석으로의 확장**을 제안합니다.

### A. 학습 과정의 행동 데이터(Behavioral Data) 트래킹
정답/오답 여부보다 문제를 푸는 **과정의 흔적**이 학생의 상태를 더 정확히 대변합니다.
* **수정 및 지우개 자국 분석**: 스캔 이미지에서 지우개로 지운 흔적, 덧쓴 자국 등을 AI 시각 모델로 감지하여 '망설임', '개념의 혼동' 정도를 측정.
* **메타인지(Self-Confidence) 지표**: 플랫폼 내에서 문제를 푼 직후(채점 전), 학생 스스로 "이 문제의 정답을 확신하는가?"를 3점 척도로 입력하게 함. 정답인데 확신이 낮으면 '찍어 맞춤', 오답인데 확신이 높으면 '잘못된 개념 고착화'로 심층 분석.
* **풀이 시간 및 체공 시간 트래킹(디지털 환경 시)**: 특정 문제에 머무른 시간을 통해 문해력 부족인지 계산 지연인지 원인을 좁힘.

### B. 지식 추적 (Knowledge Tracing) 및 오류 구조화
* **초미세 스킬(Micro-skill) 지식 그래프**: '이차방정식'이라는 큰 단원 단위가 아닌, '이차방정식 근의 공식 적용', '복잡한 식의 전개' 등 미세 스킬로 쪼개어 지식 그래프를 매핑합니다. 학생이 틀렸을 때 이전 단계의 어떤 미세 스킬 누락에서 기인했는지 역추적합니다.
* **오류 유형(Error Taxonomy)의 자동 분류**: AI가 학생의 풀이 과정을 분석하여 오답의 원인을 [개념 미숙 / 단순 연산 실수 / 문해력(조건 누락) / 논리적 비약] 4가지로 자동 태깅합니다. 이는 후속 맞춤 학습 플랜의 종류(반복 연산 훈련 vs 기본 개념 인강 시청)를 결정짓는 핵심 지표가 됩니다.

### C. 망각 곡선 기반의 예방적(Predictive) 분석
* 에빙하우스의 망각 곡선 모델을 도입하여, 학생이 과거에 완벽히 이해했던 개념이라도 복습하지 않으면 취약해질 시점을 AI가 예측합니다.
* 리포트에 "과거 극복했던 [삼각비] 개념, 지금 복습하지 않으면 잊어버릴 확률이 80%입니다"와 같은 선제적 컨설팅을 제공하여 학부모의 신뢰를 얻습니다.

## 2. 레거시(과거) 데이터 대량 마이그레이션 및 활용 방안

기존 학생들의 방대한 과거 데이터(CSV, PDF, 이미지 등)를 일일이 하나씩 리포트로 생성하는 것은 현실적으로 불가능하며 시스템 과부하를 초래합니다. 핵심은 **'과거 리포트의 생성'이 아니라 '과거 데이터를 통한 현재 메타 프로파일(Meta-Profile)의 완성'**입니다.

### A. 메타 프로파일 '타임머신' 백그라운드 학습 (Batch Ingestion Engine)
관리자(교사) 대시보드에 **[히스토리 일괄 업로드]** 기능을 제공합니다.
* **작동 원리**: 학생의 과거 데이터가 담긴 폴더(이미지, PDF 믹스)를 업로드하면, 시스템은 개별 리포트를 렌더링하지 않습니다.
* 대신 백그라운드 워커가 과거 날짜순으로 데이터를 순차적으로 파싱(Parsing)하며 AI를 통해 학생의 **'메타 프로파일(취약점, 학습 성향 등)의 내부 상태값'만 누적 업데이트**합니다. 처리 완료 시, 해당 학생은 마치 플랫폼을 수개월 사용해온 것과 동일한 수준의 정밀도 높은 현재 프로파일을 갖게 됩니다.

### B. LLM 기반 비정형 데이터 정규화 (Data Extraction)
기존 교사가 수기로 작성했거나 줄글로 된 과거 리포트(PDF/Word) 처리 방법입니다.
* 롱컨텍스트(Long-context) 처리가 가능한 LLM(예: Gemini 1.5 Pro)에 특정 학생의 1년 치 과거 리포트 PDF를 한 번에 던져줍니다.
* 프롬프트를 통해 "이 학생의 지난 1년간의 학습 태도 변화, 고질적인 수학적 약점, 교사와의 래포 형성 등을 정형화된 JSON 형태로 요약 추출하라"고 지시합니다. 이를 통해 수만 자의 비정형 텍스트를 시스템이 이해할 수 있는 정형 데이터로 즉시 변환하여 이식합니다.

### C. RAG(검색 증강 생성) 기반의 '과거 기억 서랍' (비용 효율화)
모든 과거 문제지 풀이 이미지를 정밀 분석하는 것은 컴퓨팅 리소스(비용) 낭비입니다.
* **Lazy Loading 접근**: 과거 데이터 중 방대한 이미지나 텍스트 원본은 벡터 DB에 임베딩하여 '보관'만 해둡니다.
* **적시 활용**: 학생이 현재 시점에서 특정 개념(예: '함수')에서 오답률이 급증할 때, AI가 백그라운드에서 해당 학생의 과거 데이터를 RAG 방식으로 검색합니다.
* "이 학생은 1년 전 일차함수 때도 그래프 개형을 그리는 것을 어려워했습니다. 이 부분이 근본 원인일 수 있습니다."라고 현재의 분석 코멘트를 보강하는 **참고용 장기 기억**으로만 활용합니다.

### D. 정량 데이터 전용 표준 CSV 마이그레이션
가장 확실한 정량 지표(과거 시험 점수, 단원별 정답률, 출석률)는 시스템이 제공하는 **표준 마이그레이션 CSV 템플릿**에 맞춰 교사가 엑셀로 업로드하게 합니다.
* 이 데이터는 분석 과정을 거치지 않고 곧바로 DB에 인서트되어, 학부모가 보는 플랫폼의 '연간 성장 그래프(Annual Growth Story)'에 즉시 과거 데이터 궤적으로 시각화됩니다.

## 3. 현재 개발 현황과의 비교 검증 (2026-05-22 업데이트)

제안서의 훌륭한 비전 대비, 현재 우리 시스템은 다음과 같이 기반을 다졌습니다:
- **Phase 1~2 달성**: AI 분석 코어에 '5가지 관점(초미세 스킬 및 메타인지)' 프롬프트를 이미 이식 완료하여 '결과 중심'에서 '과정 중심' 분석으로 패러다임 전환을 이뤄냈습니다.
- **RBAC 보안 (PR #40)**: 교사와 학생의 계정 권한이 엄격히 분리되고 미들웨어 최적화가 이뤄져, 향후 학생 본인이 직접 자신의 메타인지를 입력하거나(A안) 레거시 데이터를 일괄 업로드(B안)할 때 필요한 권한 뼈대가 완성되었습니다.
- **향후 과제 (Phase 4)**: RAG 기반의 '과거 기억 서랍' 기능 및 비정형 데이터 정규화 배치가 로드맵 상의 다음 대형 마일스톤(Phase 4: ML 엔진 고도화)으로 배정되어 대기 중입니다.
